# AGENTS.md

## Project Instructions

- Inspect the repository before making changes and follow existing patterns.
- Make the smallest complete change that satisfies the request.
- Preserve user changes and never revert unrelated work.
- Prefer `rg` for searching files and text.
- Use structured parsers and existing project tools when available.
- Add or update focused tests when behavior changes.
- Run the most relevant validation command before finishing when feasible.
- Report what changed, what was verified, and any remaining risk.
- Whenever you modify the code, make sure to update docs/arch.md so that it stays aligned with the current project structure and implementation.

## Git Safety

- Do not run destructive git commands unless explicitly requested.
- Do not commit changes unless explicitly requested.
- Summarize staged changes from the git index only when preparing commit messages.

## Code Style

- Prioritize human readability above all else
- All text must be written in English
- Only use ASCII characters
